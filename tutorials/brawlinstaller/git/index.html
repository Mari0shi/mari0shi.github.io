BrawlInstaller Git Tutorial

Requires

>       BrawlInstaller

>       VSCodium

o   This is recommended if you want a GUI, I personally use VSCodium but other software alternatives exist. You can technically do everything described here without it.

>       Git for Windows

 

This guide assumes you are running your build on Dolphin. Wii users will follow very similar steps except for some exceptions that are listed as they come up.

Installation

1.     Download and Install the latest version of BrawlInstaller.

a.     Either run the setup executable, or extract the .zip file to a folder of your choice

2.     Download and Install VSCodium for Windows

a.     You'll probably want one of these three options.

                                                                                                             i.      User Installer only installs for the user you're logged into (Recommended)

                                                                                                           ii.      System Installer installs for every user on the computer

                                                                                                         iii.      .zip is a compressed portable installation

b.    Launch the installation program and press next until you get to this screen

                                                                                                             i.      Make sure "Add to PATH (requires shell restart)" is checked, it should be by default. The rest of the options are pretty self-explanatory and up to you

c.     Go ahead and install the program, then close the installer and don't launch it for now

3.     Download and Install the latest version of Git for Windows

a.     I highly advise selecting the standalone installer unless you know what you're doing. Without it, you may not be able to run git commands system-wide

Select the correct Standalone Installer for your computer's architecture

b.    Launch your installer and continue until you get to this screen

                                                                                                             i.      Make sure Git LFS is checked, it should be by default

                                                                                                           ii.      Unless you already have some sort of Bash installed onto windows, I recommend just keeping these options the way they are.

c.     Continue until you get to this screen

                                                                                                             i.      If you're choosing to use VSCodium as this guide recommends, click on the drop-down and change the option to VSCodium, so it looks like this

d.    This should be your next screen

                                                                                                             i.      Functionally, you can just click next, but I personally like setting the initial branch to "main".

                                                                                                           ii.      Go ahead and press next

                                                                                                         iii.      Keep this setting on the recommended option (the middle) unless you know why you're changing it.

                                                                                                         iv.      Press Next

e.     For every page after this, it's fine to just keep the default settings. The installer will begin and complete

                                                                                                             i.      Uncheck "View Release Notes" unless you really want to view them, for some reason

                                                                                                           ii.      Press Finish

f.       You now have BrawlInstaller, VSCodium, and Git for Windows installed onto your computer.

Build Setup

>       If you already have a build set up without Git, I highly recommend starting from scratch.

>       For the purposes of this tutorial, I am going to be using the latest Project+EX Dolphin/Netplay build as a base.

o   If you're editing REX, the steps should be pretty much the same, but note that because of how humongous it is Git will take a long time to process changes. It's entirely possible though!

Setting up your repository

A repository, in this context, is just a folder with extra features. As you get to know more about git, you'll realize its other uses.

1.     Create a folder somewhere sensible on your computer, and set it up like so

📁My Build

├── 📁SD

└── 📁Load

 └── 📁Textures

└── 📁RSBE01

 

The view from "My Build" (Or whatever you chose to call it) should look like this

 

2.     Synchronize your Dolphin SD Card and your SD folder

>       This can also be done with ImDisk, or AIM Image tool, but I'm going to be using Dolphin. For Wii users, you'll have to erase your build from your SD Card and manually copy it over any time you want to update

                                                                             I.            Open up Dolphin and click on "Config"

  

                                                                           II.            Navigate to "Paths" and change your "Load Path:" to the "Load" folder you created earlier.

 

                                                                        III.            Navigate to "Wii" and set your "SD Sync Folder" to the SD folder you created earlier

>       Press "Unpack SD Card Now"

>       Click "Yes" when Dolphin warns you the folder's contents will be deleted. It should be empty anyways.

>       Dolphin will now copy your base build's contents to your SD folder. You should be able to see your folder populate

3.     Copy your base build's HD Textures into your repository

>       If they aren't available separately (like P+ or P+EX), they're typically in User -> Load -> Textures -> RSBE01

4.     Initialize your build further with BrawlInstaller

>       Open up BrawlInstaller and set your build path to your build's main folder

>       Also set your HD Texture Path to Load -> Textures -> RSBE01 from earlier

>       Click Apply, and BrawlInstaller should recognize some things

>       Set the preset to your build, unless the build is already pre-configured

 

Make sure to hit "Save"

>       Your build is now ready to be modified using BrawlInstaller, but before you make any changes, you'll want to mark this as your first commit in git.

 

Creating your first commit

1.     Go back to your repository and right-click to select "Open with VSCodium"

>       When prompted, go ahead and trust the authors, since that's you

 

2.     Click "Source Control" (The branch-looking icon) and then click "Initialize Repository"

>       Your sidebar should now look something like this.

>       Type "Initialize Repository" in the "Message" Box

 

5.     Press Ctrl+` and Type in terminal:

 

git config user.email "YOUR EMAIL OR SOMETHING ELSE HERE"

git config user.name "YOUR NAME OR SOMETHING ELSE HERE"

 

Then, press enter

 

>       These commands will set your username and email contact so git knows who authored the changes. This makes more sense in a collaborate setting, as you could imagine.

>       We can't skip this step, as git requires some sort of contact information for making a commit

>       Click on "Commit", a warning should appear saying there are no staged changes to commit

>       For the purpose of this commit, click Yes (DO NOT CLICK ALWAYS UNLESS YOU KNOW WHY YOU ARE DOING THAT)

We're doing this to have it Initialize as fast as possible. Adding this many files at once takes a while.

>       This will take time depending on how many files your build has (Usually a lot, as you can see here)

>       I encourage you to take this time to gather any mods, fighter packages, or music you'd like to add to your build

>       At some point, your sidebar should look like this

>       You've successfully initialized a git repository for your build!

>       To play your build, press "Pack SD Card" in your dolphin build, then launch as you would normally

>       This will let you track any change you make. As an example, I'm going to be installing Mega Man to my P+EX build

Using Git to track changes

>       This is going to be a simplification of a few good uses git has for being able to effectively Un-Do things in Brawl mod builds

>       Please read up more on how to use git to its fullest here

>       https://www.w3schools.com/git/default.asp

 

1.     Making a change to your build via BrawlInstaller

>       I'm going to install Mega Man to my build

 

2.     Tracking my addition in VSCodium

>       Open up VSCodium again, and have it opened to your repository

>       You'll notice the "Changes" bar has been populated again, these are all the file Additions, Removals, and Changes that installing the Mega Man fighterpackage made to my build

>       At this point, I would test out Mega Man, and see if he breaks my build. If he breaks my build, I'm gonna press the "Discard All Changes" button

 

>       When prompted, I'm gonna discard all of the files, as I want my build to be exactly the same as it was before I added Mega Man

 

Rolling back to a previous commit

>       If I want to roll back to a previous commit, because my latest commit broke it, I can also hard reset my repository back to that commit.

Oh no! Deleting Mario broke my build!

 

 

>       I'm going to copy the commit hash of the commit I want to roll back to permanently, as this cannot be undone unless you have a remote repository to pull from (like from github)

>       And I'm going to type

git reset --hard <commit hash I copied earlier>

 

in the terminal and press enter

>       This will revert my build back to the commit I labeled as "Import Mega Man"

 

I am aware that this is not best practice when it comes to git, but it's what I do when making changes on my build. I'm really just using git as an undo button.

 

EXAMPLES OF GOOD THINGS TO MARK AS COMMITS

>       Fighter Installations

>       Costume set installations

>       Stage Installations

>       Music set Installations

It isn't a very good practice to mark "every little thing" as a commit. Make your change via BrawlInstaller, test it out, and if you like it then mark it as a commit.
